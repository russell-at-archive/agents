# DDD Mastery: From Macro to Micro

Welcome to the Domain-Driven Design (DDD) Mastery guide. This series is designed
to take you from a high-level strategic understanding of DDD down to the
granular details of implementation.

## The Three Levels of DDD

DDD is most effective when applied across three distinct levels of abstraction:

1. [Macro Strategy](./01-macro-strategy.md): The "Why" and "Where"
   - Focus: Strategic Design, Bounded Contexts, and Ubiquitous Language.
   - Goal: Aligning software architecture with business domains and subdomains.

2. [Meso Workflow](./02-meso-workflow.md): The "What" and "How"
   - Focus: Tactical Design patterns and Architectural boundaries.
   - Goal: Defining the internal structure of a Bounded Context using Hexagonal
     or Clean Architecture.

3. [Micro Implementation](./03-micro-implementation.md): The "How" in Code
   - Focus: Implementation patterns, Aggregates, Entities, and Value Objects.
   - Goal: Building a rich, expressive domain model that encapsulates complex
     business logic.

## Roadmap

To become an expert, follow the documents in order. Each document contains core
principles, best practices, and actionable patterns.
