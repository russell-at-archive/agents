# CDD Micro: The Implementation

This guide covers the technical details of implementing and verifying
Contract-Driven Design (CDD) at the code level.

## Example: OpenAPI Specification (REST)

```yaml
# openapi.yaml
openapi: 3.0.0
info:
  title: Order Service API
  version: 1.0.0
paths:
  /orders/{orderId}:
    get:
      summary: Get order details
      parameters:
        - name: orderId
          in: path
          required: true
          schema:
            type: string
            format: uuid
      responses:
        '200':
          description: OK
          content:
            application/json:
              schema:
                $ref: '#/components/schemas/Order'
components:
  schemas:
    Order:
      type: object
      properties:
        id: { type: string, format: uuid }
        total: { type: number }
        status: { type: string, enum: [PENDING, COMPLETED] }
```

## Example: Consumer-Driven Contract (Pact)

```typescript
// Consumer-side test (e.g., in Jest)
describe('Order Service Client', () => {
  it('fetches an order by ID', async () => {
    await provider.addInteraction({
      state: 'an order exists',
      uponReceiving: 'a request for an order',
      withRequest: { method: 'GET', path: '/orders/123' },
      willRespondWith: {
        status: 200,
        body: { id: '123', total: 100, status: 'PENDING' }
      }
    });
    const order = await apiClient.getOrder('123');
    expect(order.id).toBe('123');
  });
});
```

## Contract Verification in CI/CD

Verification must be automated within the CI/CD pipeline.

### Provider Verification

- **Static Analysis:** Use tools like `spectral` to lint OpenAPI files.
- **Runtime Testing:** Use `Dredd` to test a running provider.
- **Mock Verification:** Use `Pact Verifier` to run consumer tests.

### Consumer Verification

- **Contract Generation:** Consumer tests generate the Pact contract.
- **Pact Broker:** Contracts are uploaded to a central "Broker."
- **Can-I-Deploy:** Use `pact-broker can-i-deploy` to check compatibility.

## Common Pitfalls

- **Contract Drift:** Forgetting to update tests when the contract changes.
- **Massive Contracts:** Defining internal implementation details.
- **Lack of Collaboration:** Writing contracts in isolation.
