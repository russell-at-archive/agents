# Prompt Engineering

Prompt Engineering is the discipline of optimizing inputs to Large Language
Models (LLMs) to achieve specific, high-quality, and reliable outputs. In 2026,
this has evolved from simple "prompting" into a tiered architectural approach:
**Macro**, **Meso**, and **Micro**.

## Learning Path

This series of documents is designed to take you from high-level system strategy
to granular implementation techniques.

1. **[01-macro-strategy.md](01-macro-strategy.md)**: System & Environment.
   Focuses on Context Engineering, RAG, and Agentic Workflows.
2. **[02-meso-workflow.md](02-meso-workflow.md)**: Structure & Logic. Focuses on
   the TCOF framework, reasoning scaffolds, and thinking management.
3. **[03-micro-implementation.md](03-micro-implementation.md)**: Syntax &
   Precision. Focuses on few-shot examples, XML delimiters, and success rubrics.

## Core Philosophy

- **Structure beats length**: A well-structured prompt is more effective than a
  long one.
- **Context is king**: Managing the context window is as important as the
  prompt itself.
- **Iterate with Evals**: Prompting is an empirical science; always test against
  a "golden set" of expected outputs.
