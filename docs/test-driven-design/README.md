# Mastery of Test-Driven Design (TDD)

Welcome to your journey toward becoming an expert in **Test-Driven Design**.
This curriculum is structured from the macro (high-level philosophy) to the
micro (low-level implementation).

## Learning Path

1. **[The Macro Manifesto](./01-macro-manifesto.md)**
   - Start here to understand the *why* and the foundational pillars (Red,
     Green, Refactor, Design).
   - Learn how TDD acts as a design tool rather than just a testing
     methodology.

2. **[The Meso Workflow Lifecycle](./02-meso-workflow.md)**
   - Transition to the *how*. Understand the Double-Loop TDD lifecycle and how
     to structure your tests.
   - Learn the tactical phases of moving from a failing test to a clean,
     decoupled implementation.

3. **[Micro Implementation Standards](./03-micro-implementation.md)**
   - Master the *execution*. Use patterns like AAA (Arrange, Act, Assert) and
     Given/When/Then.
   - Explore advanced implementation patterns like Mocking, Stubbing, and
     Triangulation.

## How to Use These Documents

- **Read sequentially:** Don't skip to implementation before understanding the
  philosophy.
- **Apply immediately:** Use the patterns in `03-micro-implementation.md` for
  your next feature.
- **Reference often:** Keep these open during your development lifecycle as a
  "constitution" for your work.

## Next Steps

Once you've read through these documents, the best way to master TDD is to
practice it. Start by writing a single failing test for a small utility
function and follow the Red-Green-Refactor cycle to completion.
